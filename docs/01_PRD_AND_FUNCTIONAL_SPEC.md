## 2. 문제 정의 (Problem Statement)

현재의 생성형 AI 도구(ChatGPT, Gemini 등)는 기업 환경에서 다음과 같은 구조적 한계를 가진다.

1. **사내 문서 활용의 제약**

   * 규정, 정책, 장애 이력, 업무 매뉴얼은 외부 AI에 직접 제공하기 어려움
2. **판단 근거의 불투명성**

   * 어떤 문서를 근거로 결론이 도출되었는지 추적 불가
3. **실행 통제 부재**

   * 실제 업무 실행(Jira, Slack, 문서 생성 등)에 대한 승인·책임 구조 미흡
4. **감사 및 재현 불가능**

   * 누가, 언제, 어떤 근거로 판단·실행했는지 기록 부족

➡ 이로 인해 AI는 기업에서 **참고용 도구**에 머물며,
➡ **판단과 실행을 맡길 수 있는 신뢰 가능한 시스템**으로 활용되지 못하고 있다.

---

## 3. 제품 비전 (Product Vision)

> **TRACE-AI는 기업 내부 지식을 기반으로 판단하고,
> 사람의 승인 하에 실행하며,
> 모든 과정을 추적·감사할 수 있는 엔터프라이즈 AI 에이전트이다.**

TRACE-AI는 단순한 대화형 AI가 아니라 다음을 통합한 시스템이다.

* **Reasoning Agent**: 지식 저장소 기반 근거 있는 판단
* **Action Agent**: 승인(Human-in-the-loop) 기반 통제된 실행
* **Audit-ready System**: run_id 기반 전 과정 추적 및 감사

---

## 4. 제품 설계 전제 (Product Assumptions)

### 4.1 실행 통제 전제 (핵심)

TRACE-AI는 **기업 환경에서의 책임성과 통제 가능성**을 최우선으로 설계된다.

* 승인 없는 외부 실행은 **절대 허용하지 않는다**
* 모든 실행은 **사람의 명시적 승인(Human-in-the-loop)**을 거친다
* 실행 결과는 성공/실패와 관계없이 **동일한 형식으로 기록·감사**된다
* 외부 시스템 연동 실패 시에도:

  * 판단 결과
  * 승인 이력
  * 로그 및 감사 기록
    은 유지된다

> TRACE-AI는 "실행을 대신하는 AI"가 아니라
> **"판단과 실행을 통제 가능한 형태로 정리하는 AI"**이다.

## 5. 목표 (Goals)

### 5.1 핵심 목표

1. 기업 내부 문서를 **지식 저장소**로 활용
2. 모든 판단 결과에 **근거(Evidence)** 제공
3. 위험한 실행에 대해 **사람 승인(Human-in-the-loop)** 강제
4. 모든 판단·승인·실행을 **run_id 기반으로 추적·감사**

---

### 5.2 해커톤 목표

  1. 지식 저장소 업로드 및 즉시 반영
  2. 규정 위반 감지 및 근거 제시
  3. 장애 RCA 수행
  4. 실행 계획 생성
  5. 승인 기반 실행
  6. 로그 및 감사 추적
  7. **엔터프라이즈 환경에 적용 가능한 통제 구조**

## 6. 대상 사용자 (Target Users)

TRACE-AI는 특정 직군에 한정되지 않으며,
**규정·사례·절차 기반 판단과 승인·감사가 필요한 모든 조직과 직군**을 대상으로 한다.

### 6.1 주요 사용자 범위

* **IT / DevOps / 운영**

  * 장애 RCA, 운영 절차 실행, 기술 규정 준수
* **보안 / 정보보호 / 컴플라이언스**

  * 정책 위반 감지, 점검 자동화, 감사 대응
* **법무 / 준법 / 내부통제**

  * 계약·정책 검토, 규정 준수 여부 판단, 근거 기록
* **인사(HR) / 노무**

  * 인사 규정 기반 판단, 내부 문의 대응, 승인 기록
* **경영지원 / 총무 / 운영지원**

  * 업무 절차 정리, 매뉴얼 기반 실행 지원
* **품질 / 감사 조직**

  * 기준 대비 이행 여부 점검, 감사 로그 활용

📌 공통 특징

* TRACE-AI는 **결정을 대신하지 않으며**,
  판단 정리 + 근거 제시 + 승인 지원을 제공한다.

## 2. 기능 정의 원칙

TRACE-AI의 모든 기능은 다음 공통 원칙을 따른다.

1. **근거 없는 판단 금지**
    - 모든 판단 결과는 지식 저장소 기반 근거(Evidence)를 포함한다.
2. **승인 없는 실행 금지**
    - 실제 업무 실행은 반드시 사람 승인(Human-in-the-loop)을 거친다.
3. **추적·감사 가능성**
    - 모든 판단·승인·실행은 run_id 기준으로 기록된다.
4. **직군 불변 구조**
    - IT/비IT 직군에 관계없이 동일한 기능 구조를 사용한다.

```
   ↓
로그 / 감사
```

- 모든 흐름은 run_id로 연결된다.
