## 4. 유스케이스 상세 명세

---

## UC-01 지식 저장소 문서 업로드

### 개요

관리자가 내부 문서를 지식 저장소에 등록한다.

| 항목    | 내용              |
| ----- | --------------- |
| 주요 액터 | Admin           |
| 사전 조건 | Admin 권한 보유     |
| 사후 조건 | 문서가 지식 저장소에 적재됨 |

### 기본 흐름

1. Admin이 지식 저장소 관리 화면(UI-06)에 접근한다.
2. 문서 파일을 업로드한다.
3. 시스템이 문서를 파싱·분할·임베딩한다.
4. 문서가 지식 저장소에 저장된다.
5. 업로드 결과가 표시된다.

### 예외 흐름

* 문서 파싱 실패 → 오류 메시지 표시 및 로그 기록

---

## UC-02 규정 위반 감지 수행

### 개요

사용자가 입력한 내용이 규정을 위반하는지 판단한다.

| 항목    | 내용            |
| ----- | ------------- |
| 주요 액터 | User          |
| 사전 조건 | 정책 지식 저장소 존재  |
| 사후 조건 | 위반 여부 및 근거 제공 |

### 기본 흐름

1. User가 요청 내용을 입력한다.
2. 시스템이 입력을 분석한다.
3. 정책 지식 저장소를 검색한다.
4. 위반 여부를 판단한다.
5. 근거 문서와 함께 결과를 표시한다.

### 예외 흐름

* 근거 부족 → “잠재적 위반”으로 표시

---

## UC-03 장애 RCA 수행

### 개요

장애 원인을 분석하고 가설을 제시한다.

| 항목    | 내용           |
| ----- | ------------ |
| 주요 액터 | User         |
| 사전 조건 | 장애 지식 저장소 존재 |
| 사후 조건 | RCA 결과 제공    |

### 기본 흐름

1. User가 로그 또는 장애 설명을 업로드한다.
2. 시스템이 유사 장애 사례를 검색한다.
3. 원인 가설을 생성한다.
4. 각 가설에 근거를 연결한다.
5. 우선순위가 정렬된 결과를 표시한다.

---

## UC-04 실행 계획 생성

### 개요

판단 결과를 바탕으로 실행 계획을 생성한다.

| 항목    | 내용       |
| ----- | -------- |
| 주요 액터 | User     |
| 사전 조건 | 판단 결과 존재 |
| 사후 조건 | 실행 계획 생성 |

### 기본 흐름

1. User가 “조치 실행”을 요청한다.
2. 시스템이 실행 계획(Action Plan)을 생성한다.
3. 각 단계의 위험도를 분류한다.
4. 승인 필요 단계를 표시한다.

---

## UC-05 실행 승인 처리

### 개요

책임자가 실행을 승인 또는 거부한다.

| 항목    | 내용          |
| ----- | ----------- |
| 주요 액터 | Approver    |
| 사전 조건 | 승인 대기 상태    |
| 사후 조건 | 승인 또는 거부 기록 |

### 기본 흐름

1. Approver가 승인 요청 화면(UI-02)에 접근한다.
2. 실행 단계를 검토한다.
3. 승인 또는 거부를 선택한다.
4. 코멘트를 입력한다.
5. 승인 결과가 기록된다.

---

## UC-06 승인 후 실행 수행

### 개요

승인된 단계만 실제로 실행한다.

| 항목    | 내용              |
| ----- | --------------- |
| 주요 액터 | TRACE-AI System |
| 사전 조건 | 승인 완료           |
| 사후 조건 | 실행 결과 기록        |

### 기본 흐름

1. 시스템이 승인된 단계만 선별한다.
2. 단계별 실행을 수행한다.
3. 실행 결과를 기록한다.
4. 감사 요약을 생성한다.

### 제약

* 승인되지 않은 단계는 실행되지 않는다.

---

## UC-07 실행 로그 조회

### 개요

사용자가 실행 로그를 확인한다.

| 항목    | 내용       |
| ----- | -------- |
| 주요 액터 | User     |
| 사전 조건 | 실행 완료    |
| 사후 조건 | 로그 조회 가능 |

### 기본 흐름

1. User가 로그 화면(UI-03)에 접근한다.
2. run_id 기준 로그를 확인한다.

---

## UC-08 감사 요약 조회

### 개요

관리자가 감사 정보를 확인한다.

| 항목    | 내용       |
| ----- | -------- |
| 주요 액터 | Admin    |
| 사전 조건 | 실행 완료    |
| 사후 조건 | 감사 기록 조회 |

### 기본 흐름

1. Admin이 감사 화면(UI-05)에 접근한다.
2. run 단위 감사 요약을 확인한다.

---

## 5. 유스케이스 간 관계

```
UC-01 → UC-02 / UC-03
UC-02 / UC-03 → UC-04
UC-04 → UC-05 → UC-06
UC-06 → UC-07 / UC-08
```

* 모든 흐름은 run_id로 연결된다.

---

> **TRACE-AI의 모든 유스케이스는
> 판단 → 승인 → 실행 → 감사의 책임 흐름을 따른다.**
