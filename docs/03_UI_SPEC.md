## 2. UI 설계 원칙

TRACE-AI의 UI는 다음 원칙을 따른다.

1. **판단 과정의 가시화**

   * “결과”보다 “근거와 과정”을 보여준다.
2. **실행 통제의 명확성**

   * 승인 전/후 상태가 명확히 구분되어야 한다.
3. **직군 중립성**

   * IT/비IT 직군 모두 이해 가능한 용어 사용.
4. **감사 친화성**

   * 실행 이력과 로그 접근이 용이해야 한다.
5. **환경 상태 표시**

   * 시스템 실행 환경 상태는 표시하되, 전환 기능은 제공하지 않는다.

## 5. 화면별 상세 설계

---

## 5.1 UI-01 메인 대화 화면

### 5.1.1 화면 목적

* 사용자 요청 입력
* AI 판단 결과 및 실행 계획 확인

---

### 5.1.2 화면 구성 요소

| 영역     | 구성 요소     | 설명                |
| ------ | --------- | ----------------- |
| Header | 실행 환경 상태  | 외부 연동 가능/제한 상태 표시 |
| Input  | 입력 텍스트 박스 | 자연어 요청 입력         |
| Input  | 파일 업로드    | 로그/문서 업로드         |
| Output | 판단 요약     | AI 판단 결과 요약       |
| Output | 실행 계획     | Action Plan 단계 표시 |

---

### 5.1.3 동작 시나리오

1. 사용자 요청 입력
2. 시스템 분석 시작
3. 판단 결과 표시
4. 실행 계획 생성 (필요 시)

---

## 5.2 UI-02 승인 요청 화면

### 5.2.1 화면 목적

* 위험 단계에 대한 사용자 승인 처리

---

### 5.2.2 화면 구성 요소

| 요소             | 설명           |
| -------------- | ------------ |
| Action Step 목록 | 단계별 실행 내용    |
| 위험도 표시         | 낮음 / 중간 / 높음 |
| 승인 버튼          | 승인           |
| 거부 버튼          | 거부           |
| 코멘트 입력         | 승인/거부 사유     |

---

### 5.2.3 승인 규칙 표시

* “승인되지 않은 단계는 실행되지 않습니다.”
* 승인 결과는 로그 및 감사에 기록됨

---

## 5.3 UI-03 실행 로그 화면

### 5.3.1 화면 목적

* run_id 기준 실행 흐름 확인

---

### 5.3.2 화면 구성 요소

| 요소        | 설명                  |
| --------- | ------------------- |
| run_id 표시 | 현재 요청 식별자           |
| 로그 타임라인   | 단계별 로그              |
| 상태 표시     | INFO / WARN / ERROR |

---

### 5.3.3 필터 기능 (선택)

* 단계별 필터
* 로그 레벨 필터

---

## 5.4 UI-04 근거(Evidence) 화면

### 5.4.1 화면 목적

* 판단 근거 문서 확인

---

### 5.4.2 화면 구성 요소

| 요소     | 설명         |
| ------ | ---------- |
| 근거 목록  | 참조된 문서 리스트 |
| 문서명    | 문서 식별      |
| 발췌 문장  | 근거 텍스트     |
| 유사도 점수 | 선택         |

---

### 5.4.3 사용자 가치

* “왜 이 판단이 나왔는지”를 직접 확인 가능

---

## 5.5 UI-05 감사 요약 화면

### 5.5.1 화면 목적

* 실행 결과 및 감사 정보 요약 제공

---

### 5.5.2 화면 구성 요소

| 요소    | 설명         |
| ----- | ---------- |
| 실행 요약 | 판단 결과 요약   |
| 승인 내역 | 승인자, 시간    |
| 실행 결과 | 성공/실패      |

---

## 5.6 UI-06 지식 저장소 관리 화면 (Admin)

### 5.6.1 화면 목적

* 지식 저장소 문서 관리

---

### 5.6.2 화면 구성 요소

| 요소     | 설명        |
| ------ | --------- |
| 문서 업로드 | 파일 업로드    |
| 문서 목록  | 업로드된 문서   |
| 메타데이터  | 유형/버전/소유자 |
| 상태     | 처리 완료/실패  |

---

### 5.6.3 접근 제어

* 관리자(Admin) 전용 화면

---

## 6. 공통 UI 요소

### 6.1 실행 환경 상태 표시

* 표시 예시:

  * “외부 연동 가능”
  * “외부 연동 제한”
* **전환 기능 없음**
* 단순 상태 정보 제공

---

### 6.2 run_id 표시

* 모든 화면에서 현재 run_id 확인 가능
* 로그/감사 화면과 연결
