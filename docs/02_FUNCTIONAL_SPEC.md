## 2. 기능 정의 원칙

TRACE-AI의 모든 기능은 다음 공통 원칙을 따른다.

1. **근거 없는 판단 금지**
    - 모든 판단 결과는 지식 저장소 기반 근거(Evidence)를 포함한다.
2. **승인 없는 실행 금지**
    - 실제 업무 실행은 반드시 사람 승인(Human-in-the-loop)을 거친다.
3. **추적·감사 가능성**
    - 모든 판단·승인·실행은 run_id 기준으로 기록된다.
4. **직군 불변 구조**
    - IT/비IT 직군에 관계없이 동일한 기능 구조를 사용한다.

---

## 3. 기능 구성 개요

TRACE-AI는 다음 **5대 핵심 기능 영역**으로 구성된다.

1. 지식 저장소 관리 기능
2. 규정 위반 감지 기능
3. 장애 RCA(원인 분석) 기능
4. 업무 프로세스 실행 기능
5. 승인·로그·감사 공통 기능

---

## 4. 기능 상세 정의

---

## 4.1 지식 저장소 관리 기능

### 4.1.1 기능 설명

기업 내부 문서(규정, 정책, 장애 리포트, 매뉴얼 등)를

AI가 활용 가능한 **지식 저장소**로 등록·관리하는 기능이다.

---

### 4.1.2 세부 기능 목록

| 기능 ID | 기능명 | 설명 |
| --- | --- | --- |
| F-01-01 | 문서 업로드 | 관리자(Admin)가 문서를 업로드 |
| F-01-02 | 문서 파싱 | 문서에서 텍스트 추출 |
| F-01-03 | 문서 분할 | 의미 단위로 문서 분할 |
| F-01-04 | 임베딩 생성 | 텍스트를 벡터로 변환 |
| F-01-05 | 지식 저장소 적재 | Vector DB에 저장 |
| F-01-06 | 메타데이터 관리 | 문서 유형/버전/소유자 관리 |

---

### 4.1.3 지원 문서 유형 (MVP)

- PDF
- TXT
- Markdown
- CSV (선택)

---

### 4.1.4 기능 제약

- 일반 사용자는 업로드 불가
- 관리자(Admin)만 수행 가능
- 업로드 이력은 로그/감사 대상

---

## 4.2 규정 위반 감지 기능

### 4.2.1 기능 설명

입력된 문서, 요청 내용, 실행 계획을

**정책/규정 지식 저장소**와 비교하여

위반 가능 항목을 탐지하는 기능이다.

---

### 4.2.2 세부 기능 목록

| 기능 ID | 기능명 | 설명 |
| --- | --- | --- |
| F-02-01 | 입력 분석 | 사용자 입력 구조화 |
| F-02-02 | 규정 검색 | 정책 지식 저장소 검색 |
| F-02-03 | 위반 판단 | 규정 위반 여부 판단 |
| F-02-04 | 근거 제시 | 위반 조항 및 문장 제공 |
| F-02-05 | 수정 권고 | 개선 방향 제안 |

---

### 4.2.3 출력 정보

- 위반 여부 (Yes / No / Potential)
- 위반 조항 목록
- 근거 문장(Evidence)
- 수정 권고사항

---

### 4.2.4 대상 직군 예시

- 보안 / 컴플라이언스

- HR / 노무

## 4.3 장애 RCA (Root Cause Analysis) 기능

### 4.3.1 기능 설명

로그, 장애 리포트, 운영 기록을 분석하여

**장애 원인 가설을 도출**하고

과거 사례 기반 근거를 제시하는 기능이다.

---

### 4.3.2 세부 기능 목록

| 기능 ID | 기능명 | 설명 |
| --- | --- | --- |
| F-03-01 | 로그 수집 | 로그/리포트 입력 |
| F-03-02 | 유사 사례 검색 | 장애 지식 저장소 검색 |
| F-03-03 | 가설 생성 | 원인 가설 도출 |
| F-03-04 | 근거 연결 | 과거 사례 매칭 |
| F-03-05 | 우선순위 정렬 | 가능성 높은 순서 정렬 |

---

### 4.3.3 출력 정보

- 원인 가설 목록 (Top-N)
- 가설별 신뢰도
- 근거 사례 및 문서
- 참고 조치 이력

---

### 4.3.4 대상 직군 예시

- IT 운영
- DevOps
- SRE

```
지식 저장소
   ↓
판단 기능 (규정 감지 / RCA)

   ↓
승인
   ↓
실행
   ↓
로그 / 감사
```
